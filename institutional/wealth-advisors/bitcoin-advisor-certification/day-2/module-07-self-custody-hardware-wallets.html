Module 7: Self Custody and Hardware Wallets

Goal: Understand direct ownership

Learn how private keys, hardware wallets, and backup strategies enable direct Bitcoin ownership and eliminate counterparty risk.

Private Keys and Ownership

What Are Private Keys?

A private key is a cryptographic secret that proves ownership and enables spending of Bitcoin. Whoever controls the private key controls the Bitcoin. This is the foundation of self-custody.

  • Public Key: Like a bank account number — safe to share
  • Private Key: Like the vault combination — must be kept secret
  • Seed Phrase: 12 or 24 words that can regenerate all keys — the master backup

Hardware Wallets

How Hardware Wallets Work

A hardware wallet is a dedicated device that stores private keys offline. Transactions are signed on the device itself, so the keys never touch an internet-connected computer.

  • Air-gapped security: Keys never exposed to the internet
  • PIN protection: Device requires authentication to use
  • Verification screen: Confirm transaction details on the device

Advisor Exercise: Simulated Hardware Wallet Setup

Time: 30 minutes

Walk through a simulated hardware wallet setup process:

  1. Seed Generation: Generate and record 24-word seed phrase
  2. Backup Methods: Practice writing seed on metal backup plate
  3. Verification: Confirm seed by re-entering selected words
  4. Test Transaction: Send a small amount and verify receipt

Discussion: What risks arise from human error in each step?

Client Tool: Self-Custody Checklist

  • Purchase hardware wallet from manufacturer (never secondhand)
  • Generate seed phrase in private location
  • Write seed phrase on durable medium (steel plate recommended)
  • Store backup in separate physical location
  • Never photograph or digitally store seed phrase
  • Test recovery process before depositing significant funds
  • Set up PIN and optional passphrase

Key Takeaways

  • Self-custody eliminates counterparty risk entirely
  • Hardware wallets are the gold standard for securing private keys
  • Seed phrase backup is the most critical step in self-custody
  • Human error is the primary risk — proper setup and testing mitigate this
