Imagine Money From Scratch

⏱️ 25 minutes 🎮 Interactive Build 🎯 First Principles

💭 Think: Design Money From Scratch

Forget everything you know. If you could design money from first principles, what rules would you choose?

  • 💎 Fixed supply — so no one can print more and steal your value
  • 🌐 No central controller — so one person can't change the rules
  • 🔐 You own it directly — no banks, no permission needed
  • 📖 Publicly auditable — trust math, not institutions
  • 🚫 Uncensorable — no one can stop a valid transaction
  • ⚡ Works 24/7 — money that never takes a break

Can you build a system with all six properties?
Let's try.

🔨 Build: Six Steps to Digital Money

1

Shared Ledger

Everyone sees the same list of who owns what. No one person controls it.

💭 Problem: How do we stop someone from writing fake transactions?

2

Digital Signatures

Every transaction must be signed with a secret key only you know—like a password that proves it's really you.

Try it: Sign a transaction

✓ Valid signature!
Only your secret key could create this signature.
3

Ordering

If two people write conflicting transactions at the same time, which one is "real"?

The Problem:

❌ Alice sends 10 BTC to Bob
❌ Alice sends same 10 BTC to Carol

We need a single agreed order, or Alice could double-spend.

4

Proof-of-Work

To add transactions, solve a hard puzzle. This makes cheating expensive—you'd need massive computing power.

Try mining: Find a hash starting with 0000...

5

Chain Blocks

Link each batch of transactions to the previous one. Changing old history breaks the chain—everyone notices.

Try it: Edit an old block

⚠️ Chain broken!
Blocks 3, 4, 5... all invalid now. History tampering detected.
6

Incentives

Reward miners with new coins + transaction fees. Makes securing the network profitable—so people actually do it.

💡 This aligns self-interest with network security. The more you invest in mining, the more you want the system to stay honest.

₿ This Design Exists.

It's called Bitcoin.

You just invented Bitcoin from first principles.
Every step you designed—Satoshi Nakamoto combined them in 2009.
