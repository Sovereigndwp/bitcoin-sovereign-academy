# Financially Sovereign Academy (FSA) - Curriculum Framework

## Overview
The FSA curriculum is designed for **maximum learning efficacy** through:
- **Adaptive sequencing** based on learner personas
- **Interactive practice** with immediate feedback
- **Real-world application** scenarios
- **Microlearning** format (20-35 minutes per module)
- **Competency-based progression** with mastery checks

## Learning Design Principles

### 1. **Competency-Based Progression**
- Learners advance based on mastery, not time
- Entry assessments adjust starting points
- Competency checks prevent knowledge gaps

### 2. **Active Learning**
- Every concept has interactive practice
- Immediate feedback on all activities
- "Learning by doing" not passive consumption

### 3. **Contextual Relevance**
- Real scenarios for every concept
- Multiple personas/situations represented
- Immediate practical application

### 4. **Cognitive Load Management**
- 20-35 minute modules max
- Single concept focus per section
- Progressive complexity building

### 5. **Spaced Repetition**
- Key concepts reinforced across modules
- Built-in review cycles
- Mastery verification at intervals

## Module Template Structure

Each module follows the **7-Element Learning Framework**:

```
1. Entry Assessment (2-3 questions)
2. Core Content (3-5 sections)
3. Interactive Demo/Calculator
4. Real-World Scenarios (3-5)
5. Comprehension Check (5-8 questions)
6. Action Plan (Takeaways)
7. Discussion Prompts (Optional)
```

## Persona-Based Learning Paths

### Path Optimization Logic:
1. **High Priority Modules** addressed first
2. **Foundation concepts** before advanced topics
3. **Immediate needs** before long-term planning
4. **Skill building** in logical sequence

### Example: Debt-Burdened Path
```
Module 1 (Cash Flow) → Module 5 (Debt Strategy) → Module 2 (Emergency Fund)
→ Module 4 (Credit) → Module 3 (Banking) → Remaining modules
```

## Assessment Strategy

### Entry Assessments
- **Adaptive questioning** adjusts module depth
- **Prior knowledge detection** prevents redundancy  
- **Confidence calibration** (what they think vs know)

### Competency Checks
- **Scenario-based questions** not just definitions
- **Applied knowledge** verification
- **Mistake identification** and correction

### Mastery Verification
- **Cross-module integration** questions
- **Real-world application** challenges
- **Peer teaching** opportunities
