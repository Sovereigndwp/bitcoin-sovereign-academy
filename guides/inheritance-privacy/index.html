Inheritance privacy context

When you pass keys to heirs, you also pass an inheritance of disclosure. This is the missing piece in most inheritance plans.

This guide is being written

The CoinJoin redesign that links here is shipping ahead of the guides themselves. The page is reserved so the link doesn't 404 and so you can see what's coming.

If you'd like to be notified when this guide lands, sign up below.

What this guide will cover (planned outline)

  1. Why a pure key handover is incomplete: heirs inherit a public spending history they don't understand and didn't choose.
  2. The privacy-context note: a short, plain-English document that lives alongside the key material.
  3. The four things the note has to record: which addresses have been seen by whom, which counterparties can match the family name to the holdings, what spending patterns the heirs should avoid, and which UTXOs are "clean" vs. "tagged."
  4. How to update the note over time without rewriting it from scratch.

Outcome: a one-page document that turns key inheritance into custody inheritance.
