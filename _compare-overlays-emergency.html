Section Overlays + Emergency Kit — Comparison

Two separate decisions on this page

Part 1 — Section Background Overlays

Almost-invisible orange gradient washes on ~5 sections. The diff is subtle by design — these are mostly mood-setters, not feature backgrounds.

O1 — Current (gradient overlay)

linear-gradient(135deg, rgba(247,147,26,0.05) 0%, transparent 100%)

Diagonal gradient from 5% orange tint at top-left fading to transparent at bottom-right. Visible if you look closely; ambient otherwise. §5 violation by strict reading.

O2 — Flat orange tint Recommended

Same warm-orange wash but uniform, no gradient. Reads identical to most users. Spec-compliant.

O3 — No tint (transparent / inherits page bg)

background: transparent;

Nothing. Section background equals page background. Strictest reading of "less colors." Cleanest. Trade-off: long pages can feel monotonous without visual rhythm — but BSA already breaks rhythm via section borders, dividers, and content density, so this is probably fine.

My take: O2 (flat tint) — preserves the warm visual rhythm without violating the gradient rule. O3 is more strict but probably feels too sterile across 5 sections.

Part 2 — Emergency Kit Banner

Sits near the bottom of the homepage. Per "color carries meaning" rule, red is justified here — Emergency Kit covers stuck transactions, lost access, scam recovery. Real safety signal. Question: how loud should it be?

E0 — Current (loud)

red gradient bg + red gradient pill button + 4 red shadows

Two gradients (banner bg + button bg), red box-shadow on both, 50px pill button. Highest urgency signal. Reads "DANGER" the moment you see it.

E1 — Flat red, P2 button Recommended

red flat 8% tint + 50% red border + P2 outline button (8px corners)

Keeps the red semantic (genuinely safety-related) but applies the unified system: flat tint not gradient, P2 button not pill, no box-shadow noise. Still reads "important" without screaming.
