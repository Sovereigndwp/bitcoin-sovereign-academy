# Purpose

Use this skill to design and scaffold an interactive demo that teaches one core idea clearly.

# Inputs

Concept: $0
Audience: $1
Interaction type: $2

# What to do

1. Define the learning objective in one sentence.
2. Define:
   - what the user sees first
   - what action the user takes
   - what changes on screen
   - what concept becomes clearer through that interaction

3. Keep the demo focused on one core idea.
4. Prefer simple state and obvious interactions.
5. Avoid long text explanations carrying the lesson.
6. Use interaction to reveal the concept.

7. Propose:
   - demo title
   - user flow
   - state model
   - sections or components
   - text labels
   - reflection prompt or takeaway

8. If implementation is requested, scaffold the files and component structure.

# Output requirements

Return:
- learning objective
- interaction model
- section map
- state plan
- suggested file structure
- implementation notes

# Quality bar

The demo should make the concept clearer within seconds, not minutes.
