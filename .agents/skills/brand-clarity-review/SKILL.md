# Purpose

Use this skill as a QA pass before shipping a page, lesson, or feature.

# Inputs

Target file or page: $0

# What to do

Review the target for:

1. clarity
2. repetition
3. brand consistency
4. visual rhythm
5. CTA clarity
6. audience fit
7. structure and flow
8. unnecessary jargon
9. style drift
10. weak or missing next step

Then:

- identify the top issues
- rank them by impact
- propose concrete fixes
- implement them if requested

# Output requirements

Return:
- key issues
- why they matter
- recommended changes
- revised copy or file changes if requested

# Quality bar

The output should be more useful than a generic review and should point to specific, high value improvements.
