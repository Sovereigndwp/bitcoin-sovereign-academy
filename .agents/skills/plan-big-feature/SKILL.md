# Purpose

Use this skill before major work so the implementation is deliberate and reviewable.

# Inputs

Feature or initiative:
$ARGUMENTS

# What to do

1. Define the goal clearly.
2. Break the work into phases.
3. Identify:
   - affected files
   - reusable components
   - dependencies
   - risks
   - decisions that should be made before coding

4. Propose:
   - phase 1
   - phase 2
   - phase 3
   - validation steps
   - rollback considerations if relevant

5. Keep the plan practical and executable.

# Output requirements

Return:
- structured plan
- file level impact
- open questions
- recommended starting point

# Quality bar

The plan should be specific enough to execute step by step in Warp.
