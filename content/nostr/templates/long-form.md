# Template — Long-form (kind 30023, NIP-23)

Use only when the topic genuinely doesn't fit in a thread. Roughly 600–1500 words. Markdown allowed.

## Front matter (added as event tags by `publish-approved.mjs`)

```yaml
title: <Title in sentence case. No emojis.>
summary: <One sentence. Plain language. No marketing copy.>
image: <optional URL to a hero image>
slug: <kebab-case-slug>
published_at: <ISO 8601 — auto-filled at publish time if blank>
```

## Structure

1. **Lede (50–100 words).** State the question or the misconception the post answers. No throat-clearing.
2. **3–5 named sections** with H2 headings. Each section is a sub-step or a sub-argument.
3. **One walk-through or one checklist** somewhere in the middle.
4. **Honest "what I'm not covering"** paragraph near the end. Builds trust.
5. **One link to BSA** at the end, where readers can keep going. Not three.

## Voice

- First-principles. Inform, don't convince.
- "You" sparingly. "I" sparingly. Most sentences should not need either.
- Plain language. If a sentence has more than one technical term, break it into two sentences.

## What to never put in a long form

- Price prediction.
- Affiliate links.
- "Subscribe to my newsletter" CTAs (the BSA Nostr account is the newsletter for now).
- Anything that would embarrass the reader who already knew this. Inform without condescension.
