# Template — Thread (5–8 notes)

A thread is a single idea broken into steps. If the idea doesn't actually have steps, it's a main post, not a thread.

## Skeleton

```
1/ <Headline statement. The whole point of the thread in one line.>

2/ <First step / first part of the idea.>

3/ <Second step. One concrete example or concrete mistake.>

4/ <Third step.>

5/ <Fourth step.>

6/ <Closing: one line that makes the headline land harder than at 1/.>

7/ <Optional CTA — at most one. /start/, /interactive-demos/, or a question.>
```

## Rules

- Each numbered note is its own short post (1–4 lines max).
- No "thread →" filler at the bottom of each note.
- The first line of note 1 has to make people read note 2 on its own — assume the rest of the thread doesn't load.
- End with a question or a link, never both.
- 5–8 notes total. Stop when the idea is done. Padding kills threads.

## Topics that work as threads

- A multi-step process (recover from a seed, set up multisig, verify a receive address).
- A list of questions a plan has to answer.
- A myth and its correction, with 3–4 concrete examples.

## Topics that don't work as threads

- "Bitcoin is great because X, Y, Z" — those are 3 separate posts.
- Anything that's really an article — use kind 30023 long form instead.
